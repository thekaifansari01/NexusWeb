📘 Nexus Web Documentation

Nexus Web is a zero‑friction, secure AI assistant that you can embed into any website in minutes. It automatically reads your page content, understands the context, and answers user questions using Groq's powerful LLMs — all while keeping your API keys safe.

Security first: Your Groq key is never exposed to the frontend. Nexus uses a separate, domain‑restricted key for widget communication.

🚀 Quick Start

Embed the Nexus widget on your website in just two steps.

1

Get your Nexus API Key

Sign in to the Nexus Dashboard and create a new API key. Copy the key — you'll need it in the next step.

2

Add the script tag

Paste the following code into your HTML, preferably just before the closing </body> tag.

<!-- Nexus Widget -->
<script>
  window.NexusConfig = {
    apiKey: 'YOUR_NEXUS_API_KEY', // Replace with your key
    theme: 'auto',                // 'dark', 'light', or 'auto'
    botName: 'Nexus AI',
    greeting: '👋 Hello! How can I assist you today?',
    model: 'llama3-8b-8192'
  };
</script>
<script src="https://cdn.nexus.ai/widget.js"></script>

Note: The CDN URL is a placeholder. In production, you'll host the widget bundle yourself or use a provided CDN link. For local testing, you can serve the index.js file from your own server.

✅ That's it! The widget will appear as a floating button on your site, ready to answer questions about your content.

You can customize the widget's behavior by setting the window.NexusConfig object before loading the script.

window.NexusConfig = {
  apiKey: 'nx_abc123',
  botName: 'DocBot',
  greeting: '📚 Welcome to our documentation! Ask me anything.'
};

🎨 Theming

Nexus automatically adapts to your website's theme, but you can also force a specific mode.

Auto‑detection

By default (theme: 'auto'), the widget checks:

  • Your window.NexusConfig.theme override
  • Saved user preference in localStorage
  • dark/light class on <body> or <html>
  • data-theme attribute
  • System preference via prefers-color-scheme
  • Defaults to dark

Manual control

Set theme: 'dark' or 'light' to lock the appearance. Users can also toggle the theme via the moon/sun icon in the chat header.

Custom CSS variables: You can override the widget's colors by targeting #ai-widget-root CSS custom properties. Check the Advanced section for details.

✨ Key Features

Page Context

Nexus scrapes your page content (using Readability + Markdown) to provide the AI with rich context, enabling accurate answers.

File Attachments

Users can attach images (and other files) to their messages. The AI can analyze images if the model supports vision.

SPA Support

Automatically detects page navigation in single‑page apps and refreshes the context, so the AI stays up‑to‑date.

Markdown + Syntax Highlighting

Bot responses are rendered with Markdown and code blocks are highlighted, making them readable and beautiful.

Plus: Responsive Accessible Lightweight Zero dependencies

🔧 Advanced Customization

Styling Overrides

You can customize the widget's appearance by overriding CSS variables on #ai-widget-root:

#ai-widget-root {
  --bg-panel: rgba(10, 10, 10, 0.92);
  --text-primary: #eee;
  --accent: #a855f7; /* primary color */
  --send-bg: #a855f7;
  --send-color: #fff;
}

Programmatic Control

The widget exposes a global API for advanced use cases:

MethodDescription
window.openNexus()Opens the chat panel programmatically.
window.closeNexus()Closes the chat panel.
window.toggleNexus()Toggles the chat panel.
window.refreshNexusContext()Forces a rescrape of the page content.

Example:

<button onclick="window.openNexus()">Chat with AI</button>

Self‑Hosting

If you prefer to host the widget yourself, bundle the source files (index.js and its dependencies) and serve them from your own CDN. Update the script src accordingly.

🔌 API Reference

The widget communicates with your backend proxy (which you can deploy on Vercel or any Node.js environment). The expected endpoint is POST /api/chat with the following payload:

{
  "nexusKey": "nx_abc123",      // Your Nexus API key
  "messages": [
    { "role": "system", "content": "..." },
    { "role": "user", "content": "Hello" }
  ],
  "model": "llama3-8b-8192"
}

The proxy validates the key, checks the domain against your authorized list, and forwards the request to Groq. The response follows the Groq API format.

You can find the serverless function implementation in api/chat.py (Python) – adapt it to your preferred stack.

🛠️ Troubleshooting

Widget doesn't appear

  • Check the browser console for errors.
  • Ensure window.NexusConfig.apiKey is set.
  • Verify that the widget script is loaded after the config object.

"Domain not authorized" error

  • Add your website's domain (without https:// or www.) to the Authorized Domains list in the dashboard.
  • Ensure the domain matches exactly (e.g., example.com).

"Invalid Nexus API Key"

  • Generate a new key from the dashboard and update your config.
  • Check that the key hasn't been revoked.

AI responses are not using page content

  • The widget caches page content for 24 hours. Force a refresh by calling window.refreshNexusContext() in the console.
  • Make sure your page content is not dynamically loaded after the widget initialises (SPA support is automatic, but you can trigger a refresh manually).
