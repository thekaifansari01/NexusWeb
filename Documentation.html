📘 Nexus Web Assistant v2.2.0

Nexus Web Assistant is a lightweight, embeddable AI chat widget that turns any webpage into a conversational knowledge base. It automatically scrapes your page content, understands the context, and answers user questions using Groq's ultra‑fast LLMs — all while keeping your API keys secure through encryption and domain whitelisting.

Security First — Your Groq key is never exposed.

You store your Groq API key in the encrypted dashboard vault. The widget uses a separate, domain‑restricted Nexus key for communication. Our proxy validates the domain, decrypts your Groq key, and forwards requests — your key never touches the browser.

🚀 Quick Start

Embed the Nexus widget on your website in two simple steps.

1

Get your Nexus API Key

Sign in to the Nexus Dashboard with Google. In the Nexus Keys tab, create a new key and copy it. Then, in the Settings tab, store your Groq API key — it will be encrypted and never exposed.

2

Add the script tag

Paste the following code into your HTML, preferably just before the closing </body> tag.

<!-- 1. Configure the widget with your Nexus API key -->
                                    <script>
                                    window.NexusConfig = {
                                    apiKey: 'YOUR_NEXUS_API_KEY',
                                    model: 'llama3-8b-8192',
                                    botName: 'Nexus AI',
                                    greeting: '👋 Hello! How can I assist you today?',
                                    theme: 'auto'
                                    };
                                    </script>

                                    <!-- 2. Load the widget from CDN -->
                                    <script defer src="https://cdn.jsdelivr.net/npm/nexus-web-assistant@2.2.0/dist/nexus-assistant.min.js"></script>

Note: The script loads asynchronously — it won't block your page rendering. The widget will appear as a floating chat button in the bottom‑right corner.

That's it! Your website now has an AI assistant that understands its content. Users can ask questions and get instant, context‑aware answers.

⚙️ Configuration Options

Customize the widget's behaviour by setting the window.NexusConfig object before the script loads. All properties are optional except apiKey.

Property Type Default Description
apiKey string '' Required. Your Nexus API key from the dashboard.
model string 'llama3-8b-8192' Groq model to use. Supported: llama3-8b-8192, llama3-70b-8192, mixtral-8x7b-32768.
botName string 'Nexus AI' Display name of the assistant shown in the chat header.
greeting string Default welcome Initial bot message shown when the panel opens. Supports Markdown.
systemPrompt string Default prompt Override the system instructions that guide the AI's behaviour.
theme string 'auto' 'dark', 'light', or 'auto' (detects system preference).

Example Configuration: 

window.NexusConfig = {
                                apiKey: 'nxs_abc123def456',
                                model: 'llama3-70b-8192',
                                botName: 'DocBot',
                                greeting: '📚 Welcome to our documentation! Ask me anything.',
                                theme: 'dark'
                                };

🎨 Theming

Nexus automatically adapts to your website's theme, but you can also force a specific mode or let users toggle it.

Auto‑detection

By default (theme: 'auto'), the widget checks in this order:

  1. Your window.NexusConfig.theme override
  2. Saved user preference in localStorage (persists across visits)
  3. dark/light class on <body> or <html>
  4. data-theme attribute on <html>
  5. System preference via prefers-color-scheme
  6. Defaults to dark

Manual control

Set theme: 'dark' or 'light' to lock the appearance. Users can also toggle the theme via the moon/sun icon in the chat header — their preference is saved automatically.

Pro tip: The widget also respects <meta name="theme-color"> brightness to infer your site's preferred theme.

✨ Key Features

Page Context Scraping

Uses Readability + Turndown to extract clean Markdown from your page content — headings, paragraphs, lists, code snippets, and JSON‑LD structured data. The AI answers with full context.

File Attachments

Users can attach images to their messages. The AI can analyze images if the chosen model supports vision capabilities.

SPA Support

Automatically detects navigation changes in single‑page apps (pushState / popState) and refreshes the page context — no manual intervention required.

Markdown Rendering

Bot responses are rendered with Markdown (via Marked) and code blocks are beautifully highlighted with Highlight.js. Tables, lists, and links work out of the box.

Smart Caching

Scraped page content is cached in IndexedDB with a 24‑hour TTL. Repeat visitors get instant responses without re‑scraping.

Vault‑Grade Security

Your Groq API key is encrypted with AES‑256 and stored in Firestore. The widget uses a separate Nexus key that only works with whitelisted domains.

🔐 Security Model

Nexus is built with a zero‑trust approach to API key management. Here's how your keys are protected:

1

You store your Groq API key in the Dashboard

Your key is encrypted with AES‑256 before being saved to Firestore. Only you can view or delete it.

2

You generate a Nexus API key for your frontend

This key is safe to embed in your HTML. It is domain‑restricted — it only works for domains you've whitelisted.

3

The widget sends your Nexus key to our proxy

The proxy validates the domain against your whitelist, decrypts your Groq key, and forwards the request to Groq. Your Groq key is never exposed to the browser.

Your Groq API key is never exposed to the frontend. Only you (via the dashboard) can manage it. This protects your billing and usage from being compromised.

Programmatic Control

The widget exposes a global API for advanced use cases. Call these functions from your own JavaScript:

Method Description
window.openNexus() Opens the chat panel programmatically.
window.closeNexus() Closes the chat panel.
window.toggleNexus() Toggles the chat panel open/closed.
window.refreshNexusContext() Forces a re‑scrape of the page content. Useful after dynamic content updates.

🔌 API Reference

The widget communicates with your backend proxy at POST /api/chat. The expected payload structure is:

{
                            "nexusKey": "nxs_abc123def456",
                            "messages": [
                            { "role": "system", "content": "You are a helpful assistant..." },
                            { "role": "user", "content": "What is this page about?" }
                            ],
                            "model": "llama3-8b-8192"
                            }

The proxy validates the Nexus key, checks the domain against your authorized list, decrypts your stored Groq key, and forwards the request to Groq's API. The response is returned to the widget and rendered with Markdown.

You can find the serverless function implementation in api/chat.py in the repository.

🛠️ Troubleshooting

Widget doesn't appear

  • Check the browser console for errors (F12 → Console).
  • Ensure window.NexusConfig.apiKey is set and valid.
  • Verify the CDN script URL is correct: https://cdn.jsdelivr.net/npm/nexus-web-assistant@2.2.0/dist/nexus-assistant.min.js

"Domain not authorized" error

  • Add your website's domain to the Authorized Domains list in the dashboard.
  • Ensure the domain matches exactly — without https://, http://, or www.
  • Example: example.com, not https://example.com or www.example.com.

"Invalid Nexus API Key"

  • Generate a new key from the dashboard and update your NexusConfig.
  • Check that the key hasn't been revoked in the dashboard.
  • Ensure the key is copied correctly — it starts with nxs_.

AI responses are not page‑aware

  • Wait a moment — the widget scrapes the page after loading. Check the console for "Page context scraped" logs.
  • For SPAs, navigation changes trigger a re‑scrape automatically.
  • You can force a re‑scrape with window.refreshNexusContext().
